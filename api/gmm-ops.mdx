Operations for performing probabilistic inference with Gaussian mixture models.

## GMMContainer

A dataclass that holds Gaussian mixture model parameters and provides basic utilities.

```python
from gmbp.gmm_ops import GMMContainer
import numpy as np

gmm = GMMContainer(
    weights=np.array([0.3, 0.7]),
    means=np.array([[0.0, 0.0], [2.0, 3.0]]),
    covs=np.array([np.eye(2), np.eye(2) * 0.5]),
    var_names=["x", "y"]
)

print(f"Components: {gmm.n_components}")
print(f"Dimensions: {gmm.dim}")
gmm.normalize()  # Ensure weights sum to 1
```

### Attributes

<ParamField path="weights" type="np.ndarray" required>
  Component weights of shape `(K,)`. Must be non-negative.
</ParamField>

<ParamField path="means" type="np.ndarray" required>
  Component means of shape `(K, D)` where K is the number of components and D is dimensionality.
</ParamField>

<ParamField path="covs" type="np.ndarray" required>
  Component covariance matrices of shape `(K, D, D)`.
</ParamField>

<ParamField path="var_names" type="List[str]" required>
  List of D variable names corresponding to the dimensions.
</ParamField>

### Properties

<ResponseField name="n_components" type="int">
  Number of mixture components (K).
</ResponseField>

<ResponseField name="dim" type="int">
  Dimensionality of the mixture (D).
</ResponseField>

### Methods

#### normalize()

Normalizes weights to sum to 1. Handles numerical underflow by falling back to uniform weights.

```python
gmm.normalize()
```

## multiply_gmms

Computes the product of two GMMs defined over the **same variables**. This is the core operation for combining factors in belief propagation.

```python
from gmbp.gmm_ops import multiply_gmms

# Multiply two factors over the same variables
product = multiply_gmms(gmm1, gmm2)

print(f"Result has {product.n_components} components")
```

### Parameters

<ParamField path="gmm1" type="GMMContainer" required>
  First GMM factor.
</ParamField>

<ParamField path="gmm2" type="GMMContainer" required>
  Second GMM factor. Must have the same variable scope as `gmm1` (order can differ).
</ParamField>

### Returns

<ResponseField name="product" type="GMMContainer">
  The product GMM with `K1 * K2` components before any reduction. Variable order matches `gmm1`.
</ResponseField>

<Note>
  The product of two Gaussians is computed using the canonical (precision) form:
  - Precision: `P3 = P1 + P2`
  - Mean: `μ3 = Σ3 @ (P1 @ μ1 + P2 @ μ2)`
  - Weight: `w3 ∝ w1 * w2 * N(μ1; μ2, Σ1 + Σ2)`
</Note>

<Warning>
  If variable names differ but have the same set, the function automatically permutes `gmm2` to match `gmm1`'s ordering.
</Warning>

## condition_gmm

Conditions a joint GMM on observed variables given evidence. Supports both hard (point) and soft (GMM) evidence.

```python
from gmbp.gmm_ops import condition_gmm, Evidence
import numpy as np

# Hard evidence: observe y = 1.0
evidence = Evidence.from_hard(np.array([1.0]))
posterior = condition_gmm(
    joint_gmm=joint_xy,
    obs_vars=["y"],
    evidence=evidence
)

print(f"Posterior over: {posterior.var_names}")  # ['x']
```

### Parameters

<ParamField path="joint_gmm" type="GMMContainer" required>
  Joint GMM over all variables.
</ParamField>

<ParamField path="obs_vars" type="List[str]" required>
  List of variable names to condition on. Must be a subset of `joint_gmm.var_names`.
</ParamField>

<ParamField path="evidence" type="Evidence" required>
  Evidence for the observed variables. Can be constructed via:
  - `Evidence.from_hard(values)` for point observations
  - `Evidence.from_gmm(gmm_container)` for soft/uncertain observations
</ParamField>

### Returns

<ResponseField name="posterior" type="GMMContainer">
  The posterior GMM over the remaining variables (those not in `obs_vars`). Has `K_joint * K_evidence` components.
</ResponseField>

<Note>
  Conditioning uses Gaussian conditioning formulas (Kalman update):
  - Conditional mean: `μ_t | o = μ_t + Σ_to @ S^-1 @ (obs - μ_o)`
  - Conditional covariance: `Σ_t | o = Σ_tt - Σ_to @ S^-1 @ Σ_ot`
  - Innovation covariance: `S = Σ_oo + obs_cov`
</Note>

### Example: Soft evidence

```python
# Soft evidence: noisy observation of y ~ N(1.0, 0.1)
soft_evidence = Evidence(
    weights=np.array([1.0]),
    means=np.array([[1.0]]),
    covs=np.array([[[0.1]]])
)
posterior = condition_gmm(joint_xy, ["y"], soft_evidence)
```

## marginalize_gmm

Marginalizes a GMM by keeping only a subset of variables.

```python
from gmbp.gmm_ops import marginalize_gmm

# Marginalize out 'z', keep 'x' and 'y'
marginal_xy = marginalize_gmm(
    gmm=joint_xyz,
    keep_vars=["x", "y"]
)

print(f"Marginal variables: {marginal_xy.var_names}")
```

### Parameters

<ParamField path="gmm" type="GMMContainer" required>
  The GMM to marginalize.
</ParamField>

<ParamField path="keep_vars" type="List[str]" required>
  List of variable names to keep. Must be a subset of `gmm.var_names`.
</ParamField>

### Returns

<ResponseField name="marginal" type="GMMContainer">
  The marginalized GMM over `keep_vars`. Number of components remains unchanged.
</ResponseField>

<Info>
  Marginalization simply slices the means and covariances to the desired dimensions. Component weights are preserved.
</Info>

## reduce_gmm

Reduces the number of components in a GMM using weighted k-means clustering and moment matching.

```python
from gmbp.gmm_ops import reduce_gmm

# Reduce to at most 10 components
reduced = reduce_gmm(
    gmm=large_gmm,
    max_components=10,
    adaptive=False
)

print(f"Reduced from {large_gmm.n_components} to {reduced.n_components}")
```

### Parameters

<ParamField path="gmm" type="GMMContainer" required>
  The GMM to reduce.
</ParamField>

<ParamField path="max_components" type="int" required>
  Maximum number of components in the output.
</ParamField>

<ParamField path="adaptive" type="bool" default="False">
  If True, uses `adaptive_component_cap` to compute a data-driven target based on weight entropy and effective sample size.
</ParamField>

<ParamField path="min_components" type="int" default="1">
  Minimum number of components (only used when `adaptive=True`).
</ParamField>

### Returns

<ResponseField name="reduced" type="GMMContainer">
  The reduced GMM with at most `max_components` components.
</ResponseField>

<Note>
  The reduction algorithm:
  1. Clusters component means using weighted k-means
  2. Merges components in each cluster by moment matching:
     - New weight = sum of cluster weights
     - New mean = weighted average of cluster means
     - New covariance = weighted average of `(cov_i + (μ_i - μ_merged)(μ_i - μ_merged)^T)`
</Note>

## combine_independent_gmms

Combines multiple independent GMMs into a single joint GMM (tensor product).

```python
from gmbp.gmm_ops import combine_independent_gmms

# Combine independent factors P(x) and P(y) into joint P(x, y)
joint = combine_independent_gmms([gmm_x, gmm_y])

print(f"Joint has {joint.n_components} components")  # K_x * K_y
print(f"Joint variables: {joint.var_names}")  # ['x', 'y']
```

### Parameters

<ParamField path="gmms" type="List[GMMContainer]" required>
  List of independent GMMs to combine. Variable scopes must be disjoint.
</ParamField>

### Returns

<ResponseField name="joint" type="GMMContainer">
  The joint GMM with `K1 * K2 * ... * Kn` components. Variable order follows the input order.
</ResponseField>

<Info>
  The joint distribution assumes independence: `P(x, y) = P(x) * P(y)`. Each pair of components creates a joint component with block-diagonal covariance.
</Info>

## Evidence

A dataclass for representing evidence in conditioning operations. Supports both hard (deterministic) and soft (uncertain) observations.

### Constructors

#### Evidence.from_hard()

Creates hard evidence from a point observation.

```python
from gmbp.gmm_ops import Evidence
import numpy as np

ev = Evidence.from_hard(np.array([1.5, 2.0]))
print(ev.means)  # [[1.5, 2.0]]
print(ev.covs)   # [[[0, 0], [0, 0]]]
```

<ParamField path="values" type="np.ndarray" required>
  Array of observed values of shape `(D,)`.
</ParamField>

#### Evidence.from_gmm()

Creates soft evidence from a GMM.

```python
ev = Evidence.from_gmm(gmm_container)
```

<ParamField path="gmm" type="GMMContainer" required>
  GMM representing uncertain/soft observations.
</ParamField>

## adaptive_component_cap

Computes an adaptive target for mixture reduction based on weight concentration.

```python
from gmbp.gmm_ops import adaptive_component_cap

target = adaptive_component_cap(
    weights=gmm.weights,
    max_components=20,
    min_components=3
)

print(f"Adaptive target: {target} components")
```

### Parameters

<ParamField path="weights" type="np.ndarray" required>
  Component weights of shape `(K,)`.
</ParamField>

<ParamField path="max_components" type="int" required>
  Upper bound on the computed cap.
</ParamField>

<ParamField path="min_components" type="int" default="1">
  Lower bound on the computed cap.
</ParamField>

### Returns

<ResponseField name="cap" type="int">
  The computed component cap in `[min_components, max_components]`.
</ResponseField>

<Note>
  Uses the geometric mean of two metrics:
  - Effective sample size: `1 / Σ(w_i^2)`
  - Entropy support: `exp(-Σ(w_i log w_i))`
  
  This provides a conservative estimate of how many components are "active" in the mixture.
</Note>
